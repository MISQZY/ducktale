# 🦆 DuckTale — сайт Minecraft-сети

Сайт для сети Minecraft-серверов **DuckTale**, включающий главную страницу и документацию для двух серверов: **DuckBurg** (выживание) и **DuckHood** (творчество).

## 🛠 Стек технологий

| Технология | Версия | Зачем |
|------------|--------|-------|
| **Next.js** | 15.x | Основной React-фреймворк, роутинг, рендеринг |
| **Tailwind CSS** | 4.x | Утилитарные CSS-классы, стилизация |
| **Fumadocs** | 15.x | Система документации (MDX-файлы → красивые страницы) |
| **TypeScript** | 5.x | Типизация кода |

---

## 📋 Предварительные требования

Перед началом убедись, что у тебя установлено:

### 1. Node.js (v18 или новее)
- Скачай с [nodejs.org](https://nodejs.org) — выбери LTS версию
- Проверь установку: `node --version` (должна быть `v18.x.x` или выше)

### 2. npm (идёт вместе с Node.js)
- Проверь: `npm --version`

## 🚀 Установка и запуск

### Шаг 1: Распакуй архив
Разархивируй `ducktale.zip` в удобную папку.

### Шаг 2: Открой терминал в папке проекта
- **Windows**: зажми Shift + ПКМ в папке → «Открыть окно PowerShell здесь»  
- **macOS/Linux**: открой Terminal, перейди в папку (`cd /путь/до/ducktale`)

### Шаг 3: Установи зависимости
```bash
npm install
```
Это скачает все необходимые пакеты (~300–500 МБ, занимает 1–3 минуты).

### Шаг 4: Запусти сервер разработки
```bash
npm run dev
```

Открой браузер и перейди на: **http://localhost:3000**

## ✏️ Как редактировать контент

### Изменить текст на главной странице
Открой `src/app/page.tsx` — весь текст там внутри JSX-разметки.

### Добавить страницу в документацию

1. Создай файл, например `content/duckburg/gameplay/new-page.mdx`:
```mdx
---
title: Название страницы
description: Краткое описание
---

# Заголовок

Текст в формате Markdown.
```

2. Добавь имя файла в `content/duckburg/meta.json` в нужном месте:
```json
{
  "pages": [
    "index",
    "---Раздел---",
    "gameplay/new-page"
  ]
}
```

### Изменить цвета / стили
Основные CSS-переменные и тема — в `src/app/globals.css`.

## 🐛 Частые проблемы

**`Module not found: Can't resolve '.source'`**  
→ Запусти `npm run dev` — Fumadocs генерирует `.source/` автоматически при первом старте.

**Ошибки при `npm install`**  
→ Убедись, что Node.js версии 18+. Попробуй удалить `node_modules/` и запустить снова.

**Страница документации показывает 404**  
→ Проверь, что файл `.mdx` добавлен в `meta.json` своего сервера.
